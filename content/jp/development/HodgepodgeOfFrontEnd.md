## Angular またはその他のフレームワークやライブラリ、便利なブレークポイントデバッグの使用方法

参照：[how-to-debug-angular-with-vscode](https://stackoverflow.com/questions/42495655/how-to-debug-angular-with-vscode)

以前 C#を書いていたときは、通常 Visual Studio のブレークポイント機能を直感的に使用して、データフローとフレームワーク構造をすばやく理解できましたが、フロントエンドでは通常コンソールを使用してすばやく処理していました。

会社のプロジェクトで大規模な[リアクティブプログラミング Reactive Programming](https://f2e.kalan.dev/advanced/23.html#reactive-programming) / アダプターパターン Adapter Pattern のフロントエンド構造に遭遇するまでは。

NGRX と Adapter によってデータフローが非常に複雑になり、データフローを追跡するためにフロントエンドのブレークポイントが必要になりました。

数バージョン前までは、Extensions Debugger Chrome をダウンロードしなければならなかったと記憶しています。

最近、VsCode に組み込まれている JavaScript Debugger は、プロジェクト内で launch.json を設定してブラウザを起動し、フレームワークを実行するだけで、簡単にブレークポイントを使用できることを発見しました。ここに記録しておきます。
