## Angular or Other Frameworks or Libraries, How to Use Useful Breakpoint Debug

ref: [how-to-debug-angular-with-vscode](https://stackoverflow.com/questions/42495655/how-to-debug-angular-with-vscode)

In the past when writing C#, I usually intuitively used the breakpoint function of Visual Studio, and could quickly understand the data flow and framework structure. But in frontend, I usually used console to handle it quickly.

Until I encountered a large [Reactive Programming](https://f2e.kalan.dev/advanced/23.html#reactive-programming) / Adapter Pattern frontend structure in a company project.

NGRX & Adapter made the data flow super complex, and at this time, frontend breakpoints were needed to track the data flow.

I remember in several versions ago, I had to download Extensions Debugger Chrome.

Until recently, I discovered that VsCode built-in JavaScript Debugger only needs to configure launch.json in the project to start the browser, and then run the framework, and you can simply use breakpoints. Recording it here.
